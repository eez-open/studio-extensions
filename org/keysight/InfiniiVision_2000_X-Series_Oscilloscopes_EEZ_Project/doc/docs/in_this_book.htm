The first few chapters describe how to set up and get started:

  • What's New, describes programming command changes in the latest version of oscilloscope software.

  • Setting Up, describes the steps you must take before you can program the oscilloscope.

  • Getting Started, gives a general overview of oscilloscope program structure and shows how to program the oscilloscope using a few simple examples.

  • Commands Quick Reference, is a brief listing of the 2000 X-Series oscilloscope commands and syntax.

The next chapters provide reference information on common commands, root level commands, other subsystem commands, and error messages:

  • Common (*) Commands, describes commands defined by the IEEE 488.2 standard that are common to all instruments.

  • Root (:) Commands, describes commands that reside at the root level of the command tree and control many of the basic functions of the oscilloscope.

  • :ACQuire Commands, describes commands for setting the parameters used when acquiring and storing data.

  • :BUS<n> Commands, describes commands that control all oscilloscope functions associated with the digital channels bus display.

  • :CALibrate Commands, describes utility commands for determining the state of the calibration factor protection button.

  • :CHANnel<n> Commands, describes commands that control all oscilloscope functions associated with individual analog channels or groups of channels.

  • :DEMO Commands, describes commands that control the education kit (Option EDU) demonstration signals that can be output on the oscilloscope's Demo 1 and Demo 2 terminals.

  • :DIGital<d> Commands, describes commands that control all oscilloscope functions associated with individual digital channels.

  • :DISPlay Commands, describes commands that control how waveforms, graticule, and text are displayed and written on the screen.

  • :DVM Commands, describes commands that control the optional DSOXDVM digital voltmeter analysis feature.

  • :EXTernal Trigger Commands, describes commands that control the input characteristics of the external trigger input.

  • :FUNCtion Commands, describes commands that control math waveforms.

  • :HARDcopy Commands, describes commands that set and query the selection of hardcopy device and formatting options.

  • :LISTer Commands, describes commands that turn on/off the Lister display for decoded serial data and get the Lister data.

  • :MARKer Commands, describes commands that set and query the settings of X-axis markers (X1 and X2 cursors) and the Y-axis markers (Y1 and Y2 cursors).

  • :MEASure Commands, describes commands that select automatic measurements (and control markers).

  • :MTESt Commands, describes commands that control the mask test features provided with Option LMT.

  • :POD Commands, describes commands that control all oscilloscope functions associated with groups of digital channels.

  • :RECall Commands, describes commands that recall previously saved oscilloscope setups, reference waveforms, or masks.

  • :SAVE Commands, describes commands that save oscilloscope setups, screen images, and data.

  • :SBUS<n> Commands, describes commands that control oscilloscope functions associated with the serial decode bus and serial triggering.

  • :SEARch Commands, describes commands that control oscilloscope functions associated with searching for waveform events.

  • :SYSTem Commands, describes commands that control basic system functions of the oscilloscope.

  • :TIMebase Commands, describes commands that control all horizontal sweep functions.

  • :TRIGger Commands, describes commands that control the trigger modes and parameters for each trigger type.

  • :WAVeform Commands, describes commands that provide access to waveform data.

  • :WGEN Commands, describes commands that control waveform generator (Option WGN) functions and parameters.

  • :WMEMory<r> Commands, describes commands that control reference waveforms.

  • Obsolete and Discontinued Commands, describes obsolete commands which still work but have been replaced by newer commands and discontinued commands which are no longer supported.

  • Error Messages, lists the instrument error messages that can occur while programming the oscilloscope.

The command descriptions in this reference show upper and lowercase characters. For example, :AUToscale indicates that the entire command name is :AUTOSCALE. The short form, :AUT, is also accepted by the oscilloscope.

Then, there are chapters that describe programming topics and conceptual information in more detail:

  • Status Reporting, describes the oscilloscope's status registers and how to check the status of the instrument.

  • Synchronizing Acquisitions, describes how to wait for acquisitions to complete before querying measurement results or performing other operations with the captured data.

  • More About Oscilloscope Commands, contains additional information about oscilloscope programming commands.

Finally, there is a chapter that contains programming examples:

  • Programming Examples.

Mixed-Signal Oscilloscope Channel Differences

Because both the "analog channels only" oscilloscopes (DSO models) and the mixed-signal oscilloscopes (MSO models) have analog channels, topics that describe analog channels refer to all oscilloscope models. Whenever a topic describes digital channels, that information applies only to the mixed-signal oscilloscope models.
